## 何を予測するのか・前提条件

## 予測モデルの設計（メイン）

> 📝 ここに書くこと: この章が記事の中心。「1つのモデルに頼らず、性質の違う複数の見方をブレンドする」という設計思想を最初に一段落で述べてから、各モデルを数式・原理から説明する。

最終的な優勝確率は、性質の異なる3つの確率を重み付き平均した **3者ブレンド** で決める:

$$
P_{\text{blend}} = 0.4 \, P_{\text{DC}} + 0.2 \, P_{\text{Elo}} + 0.4 \, P_{\text{odds(Shin)}}
$$

（`ensemble: w_dixon_coles=0.4 / w_elo=0.2 / w_odds=0.4`）

> 📝 ここに書くこと: 用語の整理を最初にしておくと混乱が減る。「**シム(mix)**」＝試合レベルで Dixon-Coles と Elo を重み付き混合した *MixtureMatchModel* をモンテカルロに通して出す優勝確率。「**ブレンド**」＝チームレベルでその *シム(mix)* とオッズ(Shin)を 0.4/0.2/0.4 で混ぜた最終確率。乖離分析で使う「モデル」は前者（シム）を指す点に注意。// TODO: 確認（mix の DC:Elo 内部比率を実装で確認して明記。`cli.py simulate --model mix` の重み）

### Dixon-Coles（過去スコアからの統計モデル）

> 📝 ここに書くこと: 各チームの攻撃力 att・守備力 def を過去スコアから最尤推定し、試合の両チーム得点をポアソンでモデル化する古典的手法（Dixon & Coles, 1997）。ロースコア依存補正と時間減衰がキモ、という流れで。

- 得点期待値（中立地なら $\gamma$ の項は落ちる）:

$$
\log \lambda = \text{base} + \text{att}_{\text{home}} - \text{def}_{\text{away}} + \gamma \cdot \mathbb{1}[\text{home}], \quad
\log \mu = \text{base} + \text{att}_{\text{away}} - \text{def}_{\text{home}}
$$

- **ロースコア補正** $\tau(x,y)$: 独立ポアソンが過小評価しがちな (0,0)(1,0)(0,1)(1,1) を $\rho$ で補正。$\rho$ は `[-0.15, 0.15]` の範囲で同時推定。
- **時間減衰**: 各試合に $\exp(-\xi \cdot d)$（$d$＝基準日からの経過日数）の重みを掛けた**重み付き最尤推定**。$\xi = 0.0005$/日 を採用。これは `cli.py tune-xi` で2022年大会を使ってCVした結果で、**初期値の 0.0019 より小さい＝長めの記憶を好む**という結果（CV log lossが 0.0003〜0.0005 を選好）。
- 安定化: 攻撃/守備パラメータに L2 正則化（`l2 = 1.0`）、得点は `max_goals = 10` で打ち切り。現大会の試合は再フィット時に `tournament_weight = 2.0` で上乗せ。
- 推定は scipy の L-BFGS-B（解析的勾配つき）。

### Elo（チームレーティング）

> 📝 ここに書くこと: チェス由来の相対レーティング。試合結果で増減し、強さの「現在地」を表す。Dixon-Colesが攻守を分解するのに対し、Eloは1次元で直近の勢いを素早く反映する、という対比を書くと良い。

- 期待勝率（引き分けは0.5換算）: $E = \dfrac{1}{1 + 10^{-\Delta/400}}$
- 更新: $R \leftarrow R + K \cdot m(\text{得失点差}) \cdot (\text{結果} - E)$。得点差マルチプライヤ $m$ は eloratings.net 方式（2点差=1.5、3点差以上=$1.75 + (g-3)/8$）。
- **K値は大会の格で変える**: ワールドカップ **60** / 予選 **50** / 大陸選手権 **50** / 親善 **20** / その他 **35**。初期レート 1500、非中立時のホーム補正 100。
- Elo差を得点期待値に写像（モンテカルロ用）: 平均総得点 `total_goals = 2.6`、`goal_diff_per_elo = 0.0045`（Elo1点あたりの期待得失点差）、下限 `min_lambda = 0.15`。

### Shin法でブックメーカーオッズから「市場の素の確率」を抽出

> 📝 ここに書くこと: ブックメーカーの提示オッズの逆数を足すと1を超える（＝控除率/オーバーラウンド）。これを除いて「市場が織り込む確率」を取り出す。単純な比例配分ではなく Shin (1992) 法を使う理由（インサイダー取引の存在を仮定してロングショット・バイアスを補正する）に触れる。

- 比例法: $p_i = \dfrac{1/o_i}{\sum_j 1/o_j}$（単純に正規化するだけ）。
- **Shin法**: インサイダー比率 $z$ を導入し、de-margin後の確率が1に和になる $z$ を二分法で解く。実装は `proportional` と `shin` の両方を出して比較できるようにしてある（乖離分析の主役は `p_shin`）。

### 3者をなぜ混ぜるのか

> 📝 ここに書くこと: 3者の長所・短所を一段落で。Dixon-Coles＝スコアの構造を捉えるが直近の勢いに鈍い／Elo＝勢いに敏感だが攻守の質や相手依存を捨象／オッズ＝最も情報量が多いが市場心理・人気バイアスを含む。0.4/0.2/0.4 という重みの根拠（`cli.py optimize-weights` によるBrier/log lossバックテストで再最適化できる仕組みであること）にも触れる。// TODO: 確認（現行の重みが手動既定か最適化結果か）

## シミュレーション手法

> 📝 ここに書くこと: 確率を「大会を最後まで何度もプレーさせて数える」モンテカルロで出す、という全体像から。試合モデル（シム＝DC×Eloの混合）が1試合のスコアを生成し、それを大会丸ごとに積み上げる。

- **試行回数 10,000回**、乱数 `seed = 42` で再現性を担保（`simulation: n_sims=10000, seed=42`）。
- **現在状態からの継続シミュレーション**: 既に終わった試合（グループ・ノックアウト問わず）は実結果に固定し、**残り試合だけ**をモデルからサンプリングする。大会が進むほど確率は実績に引っ張られて鋭くなる。
- **トーナメント進行**:
  - グループ: 残りの総当たりをサンプリング → 勝点等で順位確定 → 各組1・2位、および全12組3位から成績上位8チームを規定スロットへ割り当て。
  - ノックアウト: R32 → R16 → QF → SF → 決勝をブラケット通りに解決。各試合は勝者を1人決めて次のラウンドへ。
- **延長・PK**: 90分が引き分けなら強度 `extra_time_factor = 0.30`（30分ぶんの得点強度）で延長をサンプル。なお引き分けなら、Elo差に応じたPK（`pk_elo_scale = 0.25`、`pk_elo_scale=0` で完全50/50）で決着。
- 1試合のスコア生成は混合モデル: 各試行・各試合でDCモデルかEloモデルかを重みで選んでスコアをサンプルする（`MixtureMatchModel`）。

## モデル vs 市場の乖離分析（記事のメイン読み物）

> 📝 ここに書くこと: ここが一番おもしろい章。自作モデル（シム）と市場（オッズShin）の優勝確率が大きく食い違ったチームを、実数とともに紹介する。乖離の定義と、なぜ食い違うのかを読み解くフレームを提示する。

- **乖離の定義**: `divergence_pts = (P_champion(シム) − P_shin) × 100`。絶対値が **3.0pt 以上**（`divergence_threshold_pts = 3.0`）でフラグ（⚠）を立てる。
- **Argentina（モデルが強気）**: モデル 約21.9% に対し市場 約10.7%、**乖離 +11.2pt**。好スタート（初戦3-0）と高いEloが効いている、と解釈できる。// TODO: 確認（公開直前に最新スナップショットの latest.json で数字を更新）
- **France（モデルが弱気）**: 市場 約19.8% に対しモデル 約7.6%、**乖離 −12.2pt**。市場は優勝本命に推すがモデルは慎重。// TODO: 確認（同上、最新値に更新）

> 📝 ここに書くこと: 解釈フレームを言語化する。乖離が出る典型要因＝(a) モデルが直近の勢い/Eloを過大評価、(b) 市場が「優勝候補ブランド」を人気で過大評価、(c) 日程の組み合わせ運（ブラケットの楽/険）。実装には乖離国を別のClaude会話に投げて定性チェックする `qualitative_prompts`（怪我・出場停止・監督交代・xGと結果の乖離をWeb検索で確認）の仕組みもあるので、その運用を紹介してもよい。

## モデル精度の検証

> 📝 ここに書くこと: 「当たった/外れた」ではなく確率予測としての良し悪しを測る、という枠組みを最初に。試合前に記録した予測（後出し禁止）だけを実結果と突き合わせる公平性ルールにも触れる。

- **Brier score**: 予測確率ベクトルと実結果のone-hotの二乗誤差和（小さいほど良い）。試合ごとに算出し平均。
- **log loss**: 実現アウトカムに割り当てた確率の負の対数（自信過剰を強く罰する）。
- **calibration**: mixモデルの全アウトカム確率をビンに分け、「予測確率 vs 実際にそうなった頻度」をプロット。対角線に乗るほど確率が信用できる。
- **モデル横断の比較**: DC単体 / Elo単体 / mix を同じ指標で並べて、ブレンドが単体に勝っているかを見る（`src/backtest.py` がDC:Eloの最適シェアやxiをCVで選ぶ仕組み）。
- 累積指標は大会進行とともに `accuracy.json` に蓄積され、サイトの精度ページで推移を見られる。
